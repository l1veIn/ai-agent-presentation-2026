# 第七章：收尾 — 量力而行的哲学 (4-5 页)

## 7.1 三论回顾

用一张统一大图呼应开场路线图：

| 理论 | 对应阶段 | 核心洞察 |
|------|---------|---------|
| 信息论 | Web 对话式 AI | Prompt = 编码，信噪比决定输出质量 |
| 控制论 | IDE AI / CLI Agent | 闭环反馈 > 开环，反馈维度越多控制越精确 |
| 系统论 | Computer Use / 编排 | 编排复杂度有刚性上限，超过阈值系统必崩 |

## 7.2 🎓 终极洞察：总熵最低原则

```
总熵 = 科研任务熵 - AI降低的熵 + AI引入的熵

最好的工作流 = 让总熵最低的组合
```

- AI 降低的熵：文献调研结构化、代码自动纠错、数据自动处理
- AI 引入的熵：学习成本、工具管理、幻觉错误、网络依赖
- 不是越多越好，而是**找到净收益最大的甜蜜点**

## 7.3 五条建议

1. **从简单开始，逐步升级** — 别一上来就编排 10 个 Agent
2. **理解能力边界** — 边界感 > 崇拜感
3. **投资 Prompt 工程** — 10 分钟好 Prompt 省 2 小时调试
4. **好奇心 yes，焦虑 no** — 掌握方法论，新工具一看就能定位
5. **AI 是放大器** — 放大的是你已有的能力，核心竞争力永远是你自己
