# 第二章：对话式 AI — 最低门槛的入口 (5-6 页)

> **叙事定位**：这是起点。听众大多已有使用经验，这一章的目标不是"教他们用"，而是帮他们**建立分析框架**——用信息论理解为什么有些 Prompt 效果好、有些不好。

## 2.1 三大对话式 AI 概览

一张精心设计的对比卡片（不是传统表格）：

| | ChatGPT (OpenAI) | Claude (Anthropic) | Gemini (Google) |
|---|---|---|---|
| **擅长** | 通用对话、代码、插件生态 | 长文本分析、细腻推理、安全性 | 多模态、Google 生态、长上下文 |
| **上下文窗口** | 128K (GPT-4o) | 200K (Claude Opus) | 1M+ (Gemini Pro) |
| **多模态** | 图像/语音/视频 | 图像/PDF | 图像/视频/音频/代码 |
| **关键差异** | 插件商店、GPTs | 思考过程可见、Artifacts | 与 Google 全家桶联动 |

- 强调：**没有"最好的"，只有"最适合当前任务的"**
- 面向 AI 学院学生可以稍微深入：不同模型的训练策略差异导致的行为差异

### 2.1.1 国产大模型矩阵 — 不可忽视的力量

> 国内 AI 生态已经非常成熟，对于科研场景（尤其是中文语料）有独特优势。

| | DeepSeek | 豆包 (字节) | 通义千问 (阿里) | 智谱 GLM | MiniMax |
|---|---|---|---|---|---|
| **定位** | 开源推理强者 | 字节全家桶集成 | 阿里生态联动 | 学术底蕴深厚 | 长上下文专家 |
| **核心优势** | 开源、推理能力强、性价比极高 | 飞书/抖音集成、多模态 | 代码能力强、工具调用 | 学术论文理解、知识图谱 | 超长上下文窗口 |
| **科研适用** | 代码/数学推理、本地部署实验 | 日常问答、多模态分析 | 代码辅助、数据分析 | 论文精读、学术写作 | 长文献综述、多文档分析 |
| **特色** | R1 思维链推理震惊行业 | 无需科学上网 | Qwen 系列开源生态 | 清华团队出品 | 海螺 AI 端产品 |

**给学生的关键信息**：
- **网络友好**：国产模型无需科学上网，在校园网环境下零门槛使用
- **DeepSeek 特别提**：R1 的思维链推理能力在数学和代码领域已接近甚至超越 GPT-4o，且完全开源可本地部署
- **实用建议**：日常快速查询用豆包/千问，深度推理用 DeepSeek，学术场景试试智谱

## 2.2 实战演示：用对话式 AI 做文献调研

- **场景**：给定一个研究方向（比如"多模态大模型的幻觉问题"），让 AI 梳理 Related Work
- **演示对比**：

```
❌ 坏 Prompt：
"帮我找一下多模态大模型幻觉的相关论文"

✅ 好 Prompt：
"你是一位计算机视觉领域的资深研究者。我正在研究多模态大模型
（如 LLaVA、GPT-4V）中的幻觉问题。请帮我：
1. 梳理 2023-2024 年该方向的 5 篇核心论文
2. 按照"问题定义→方法分类→评估基准"的结构组织
3. 指出目前的 open problems
请用学术写作风格，引用格式用 [Author, Year]"
```

- 展示两种 Prompt 的输出对比，让听众直观感受差距
- 引出理论：**为什么好的 Prompt 效果好？信息论可以解释**

## 2.3 🎓 理论彩蛋 #1：信息论 — 香农信道模型

> 这一页是整场演讲的第一个理论节点。用一张精心设计的信道模型图。

**你和 AI 之间就是一条信道：**

```
┌──────────┐    Prompt     ┌──────────┐    Response    ┌──────────┐
│          │  (编码信号)    │          │   (解码信号)    │          │
│  你 (信源) ├─────────────→│ AI (信道) ├───────────────→│ 输出 (信宿) │
│          │               │          │                │          │
└──────────┘               └────┬─────┘                └──────────┘
                                │
                           噪声源：
                        • 模糊的指令
                        • 缺失的上下文
                        • 矛盾的要求
                        • 超出训练分布
```

**三个关键映射：**

1. **信道容量 ↔ 上下文窗口**
   - 信息论告诉我们：信道容量有上限
   - AI 的上下文窗口就是信道容量——128K tokens 不代表无限理解，有效信息密度才是关键
   - 超过信道容量 → 信息丢失（"遗忘"早期对话）

2. **信噪比 ↔ Prompt 质量**
   - 好 Prompt = 高信噪比 = 有效信息占比高
   - 坏 Prompt = 低信噪比 = 充满冗余和歧义
   - **Prompt 工程的本质：最大化信噪比**

3. **编码效率 ↔ 结构化表达**
   - 人类自然语言是低效编码（冗余高）
   - 结构化 Prompt（角色+任务+格式+约束）= 高效编码
   - 这就是为什么 few-shot、chain-of-thought 有效——它们在帮 AI 校准解码器

**给 AI 学院学生的 bonus**：
- 香农第二定理（有噪声信道编码定理）的启示：存在一种编码方式，让传输错误率趋近于零——这就是为什么 prompt engineering 有理论上限，而不是无限优化的

## 2.4 对话式 AI 的天花板

- **没有持久记忆**：每次对话都是新信道（无状态）
- **无法操作外部世界**：只能生成文本，不能执行动作
- **无法验证自己的输出**：开环系统，没有反馈
- 用一张图总结：

```
对话式 AI = 开环系统

输入(Prompt) ──→ [AI] ──→ 输出(Response)
                              ↓
                          没有反馈！
                    你需要人工验证每一个输出
```
