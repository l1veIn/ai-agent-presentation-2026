一份基于 Slidev 制作的演示文稿，简要梳理 2026 年初 AI 智能体（Agents）的发展脉络。内容涵盖了 IDE 工具链的演进、OS 级控制台的探索，以及国内围绕 OpenClaw 展开的“龙虾大战”生态切片。

希望能以相对直观的方式，探讨现代工作流如何从“对话式建议”逐步过渡到“闭环自动执行”。

# 代码与审美约束
在编写 Slidev 代码时，你必须严格遵守以下 5 条极简架构原则，禁止偏离：

1. **结构原则**：入口为 `slides.md`。必须利用 Slidev 的 `src:` 语法将不同的章节拆分到独立的 Markdown 文件中（例如 `chapters/01-xx.md`），保证代码高内聚低耦合。
2. **样式原则（去 AI 味）**：全篇使用 Tailwind CSS，配色必须是高级暗黑极简风格（Dark Slate）。绝不使用花哨的霓虹发光（Neon）、夸张的跳动动画或高饱和度渐变色。边框应仅使用简洁的单线框（如 `border-slate-700`）。
3. **内容原则**：禁止出现任何类似“颠覆、神级、降维打击”等中二修辞；行文必须克制、精炼、陈述物理事实。彻底禁用所有 Emoji，全篇必须且只能使用 `Lucide` 矢量图标库（语法严格采取 `<lucide-xxx />` 防止解析失败）。
4. **排版原则（抗拒拥挤）**：单页核心论点控制在 3-4 个以内。所有发散性对比必须使用 HTML `grid grid-cols-2` 进行严格分栏布局。复杂组件禁止使用易越界的层叠绝对定位，凡超过 50 行的 SVG 或 CSS 原理图必须果断舍弃，改用 `img` 并要求人类提供或自动生成概念画作替代。
5. **构建验证**：必须确保 `package.json` 安装了 `@iconify-json/lucide`。开发完成必需运行 `npm run build` 自检，杜绝标签未闭合导致的布局溢出。
